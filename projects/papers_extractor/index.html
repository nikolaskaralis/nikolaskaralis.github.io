As of April 2013, Papers 2 has added the missing functionality that the code below was trying to solve. So this project is now obsolete and only applies to the previous versions. It will remain online in the hope that it might help some more people liberate their databases.







After experimenting for a while with the three major reference management software (Mendeley, Zotero, Papers),
I decided that the best solution for my taste is to keep my papers classified in categories and use Zotero to add the citations when writing. Mendeley is probably the most concrete platform but has a steeper learning curve.

However one major issue when using the Papers2 software to manage your papers, is the fact that you cannot export massively all the pdf files from one Collection. You do have access to the whole library and the database, but not to the collections.
I believe that this functionality is not provided by the developers, in an effort to tie you to this software and make the complete transition of your library to another reference management software difficult.


In order to resolve this issue, I wrote this software that extracts the pdf files from the Papers2 database, while preserving their classification in Collections.
It is distributed under the GNU GPL licence.

How to use it


You can either use the application file (double click to run) or if you need to adjust some of the settings, you can edit the code.
In order to run the code, you need to open a terminal, navigate to the directory where you saved it and type :
python papers_extractor.py

Ifthe path where your Papers database is located is different than the default or if you would like to change the output path, you can do so by changing the following variables respectively :
input_path
output_path

This is a quick code to get the job done, so there is no extensive Exception catching etc.
Make sure that your paths are correct.
